# VPN Service MVP

Сервис управляет WireGuard-профилями через Telegram-бота и веб-админку.

## Что уже есть

- отдельный peer и профиль для каждого устройства;
- до 10 активных устройств на пользователя;
- пробный период, который начинается при создании первого профиля;
- ручная выдача дней подписки через админку;
- отзыв отдельных устройств и блокировка пользователя;
- автоматическое отключение peers после окончания доступа;
- статистика последнего handshake и суммарного трафика;
- одноразовая выдача приватного ключа без хранения его в базе.

## Архитектура

- `bot`: Telegram-интерфейс и минутная синхронизация peers;
- `web`: FastAPI-админка на `/admin`;
- `db`: PostgreSQL;
- `wg0`: WireGuard-интерфейс на Linux-хосте.

WireGuard дает высокую скорость и хорошую стабильность, но сам по себе не гарантирует
работу при блокировке его сигнатуры. Для устойчивого публичного запуска основной
кандидат — AmneziaWG 2.0, а обычный WireGuard останется контрольным транспортом.
Backend изолирован в `app/vpn.py`, поэтому после испытаний на первом сервере транспорт
можно заменить или дополнить без переделки бота, подписок и админки.

Не существует протокола, который гарантированно переживает любые будущие блокировки.
Перед публичным запуском нужно проверить несколько транспортов на реальных российских
операторах, измерить скорость и заложить быструю ротацию IP/доменов.

## Локальный запуск без VPN-сервера

```bash
cp .env.example .env
# Оставьте VPN_BACKEND=mock и заполните BOT_TOKEN
python -m venv .venv
. .venv/bin/activate
pip install -e '.[dev]'
uvicorn app.main:app --reload
```

Админка: `http://127.0.0.1:8000/admin`. Логин и пароль берутся из `.env`.

## Развертывание на пробном Ubuntu/Debian-сервере

Требования: публичный IPv4, открытый UDP-порт `51820`, Docker Compose и домен или IP.

```bash
sudo WG_PORT=51820 bash deploy/setup-wireguard.sh
cp .env.example .env
```

Заполните `.env`:

```dotenv
BOT_TOKEN=...
ADMIN_PASSWORD=длинный-случайный-пароль
VPN_BACKEND=wireguard
WG_SERVER_PUBLIC_KEY=результат-команды-sudo-wg-show-wg0-public-key
WG_ENDPOINT=публичный-ip-или-домен:51820
```

Затем:

```bash
docker compose up -d --build
curl http://127.0.0.1:8000/health
```

Для внешнего доступа к админке поставьте перед портом `8000` HTTPS reverse proxy и
ограничьте доступ по IP/VPN. Не публикуйте админку напрямую в интернет.

## Эксплуатационные замечания

- При истечении доступа активные профили отзываются в течение одной минуты. После
  продления пользователь получает новый профиль.
- `create_all` подходит для пробного MVP. До публичного запуска нужно добавить Alembic,
  резервные копии PostgreSQL, мониторинг и отдельный контроллер VPN-узлов.
- При появлении нескольких VPN-серверов выдачу IP и операции с peers нужно перенести в
  node-agent с очередью задач и транзакционными блокировками.
